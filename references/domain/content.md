Content Refiner Module

Domain

Written Content Artifacts (Markdown, HTML, PDF-ready)

Purpose

Refine structured content into publication-ready artifacts.

⸻

Inputs
	•	specification (from Specify phase)
	•	plan (from Plan phase)
	•	existing content file(s) (optional)

⸻

Responsibilities
	1.	Improve structure and clarity
	2.	Normalize headings and hierarchy
	3.	Enforce tone and voice alignment
	4.	Convert Markdown to HTML
	5.	Generate PDF-ready output if required

⸻

Deterministic Execution Use Cases

Use code_interpreter when:
	•	Converting Markdown → HTML
	•	Generating PDFs
	•	Checking broken links
	•	Validating structure
	•	Running readability analysis

⸻

Common Constraints
	•	Tone consistency
	•	Structured heading hierarchy
	•	Proper link formatting
	•	Accessibility considerations
	•	Brand voice adherence

⸻

Expected Outputs
	•	Refined content file
	•	HTML version
	•	Optional PDF export
	•	Updated artifact_manifest.json

⸻

Reflection Focus
	•	Clarity and coherence
	•	Structural soundness
	•	Tone alignment
	•	Technical correctness
