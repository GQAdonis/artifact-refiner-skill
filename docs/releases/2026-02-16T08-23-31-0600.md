## Release Summary

This release implements the OpenSpec change `add-browser-rendering-for-htmx-react-artifacts` end-to-end, including proposal/design/spec/tasks artifacts and the full implementation of browser preview rendering support for UI and A2UI refinement outputs.

## OpenSpec Status

- Change: `add-browser-rendering-for-htmx-react-artifacts`
- Schema: `spec-driven`
- Artifacts complete: `proposal`, `design`, `specs`, `tasks`

## Implemented Capabilities

- Added deterministic browser preview planning and execution flow for `ui` and `a2ui` domains.
- Added TSX preview compilation support via `scripts/compile-tsx-preview.mjs`.
- Added browser preview renderer and screenshot/diagnostics capture via `scripts/render-preview.mjs`.
- Added shared preview utility module `scripts/lib/preview-utils.mjs`.
- Added HTMX runtime handling with offline-first local runtime and optional network mode.
- Added manifest preview metadata support (`preview.runs`, validation metadata).
- Added stronger post-execute and manifest validation checks for preview artifacts.
- Updated executor/validator agent guidance for preview generation and verification.
- Added UI preview example artifacts including preview HTML, report JSON, and screenshot image.
- Added optional MCP renderer config example and local dependency declaration for Playwright/esbuild.
